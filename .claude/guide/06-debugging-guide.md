# 06 — 硬件调试方法论

## ⚠️ 第 0 步：烧录了没？

**改完代码没点 Download = MCU 里跑的还是旧固件。** 信号脚 0V 时，先确认刚改的代码确实烧进去了。

## ⚠️ 排查第一原则：先确认供电拓扑

**USB 只供 MSPM0，电池才供电机。** "板子有电" ≠ "所有模块有电"。

| 电源来源 | 供电范围 |
|---------|---------|
| USB 5V | MSPM0 → 板载LDO → 3.3V → TB6612 VCC（逻辑） |
| 电池 7.4V | TB6612 VM（电机动力）、XL4005×2 IN |
| XL4005#1 6.0V | MG996R 舵机 |
| XL4005#2 5.0V | MSPM0 5V（非USB时）、灰度传感器 |

**排查外设问题，先逐级确认电源域**，不要跳过去查代码/信号。

## 电机不转的标准排查顺序

```
① RGB LED 闪不闪？              → 不闪 = 代码没跑/供电问题
② TB6612 VM、VCC 对 GND 正常？  → 没有 = 面包板/电源线
③ STBY、AIN1、AIN2、PWMA 用万用表一个个测
④ 电机直接碰电池转不转？        → 不转 = 电机坏了
⑤ 以上全对但 AO 无输出？        → TB6612 挂了，换一颗
```

> 本项目实际踩了①②⑤。

## TB6612 好坏判断

VM/VCC 正常 + STBY=H + AIN1/AIN2 正确 + PWMA 有信号 → AO 无输出 = **坏了**。

新模块也有良品率问题，本项目第一颗到手死。

## 舵机不动排查

```
① 红棕线接 6V+GND → 黄线碰 3.3V → 跳 = 舵机好
② 万用表 DC 档测 SIG 脚 → ~0.25V = PWM 正常，~3.1V = 异常
③ 查代码：TIMG0 是否用了 Timer 模式而非 PWM 模式
④ 查 GPIO：PA12 是否 enableOutput
```

## 排查技巧

- **电机/外设不响应** → 先确认代码在跑（加 LED 闪烁），再怀疑 GPIO 配置
- **面包板电压乱跳** → 接触不良，换孔或换杜邦线
- **大电流供电** → 尽量杜邦线直连，不走面包板电源轨
